## 技術細節

### 問題診斷過程

1. **Database ID 不匹配**
   - .env: 3002895b-b1ee-8116-b6c0-c0d960ca177b
   - 實際: ed9cef8e-72ab-48ca-9c8e-7e1f5d21d439
   - 原因：之前測試時寫錯了

2. **欄位名稱不一致**
   - 程式碼使用 emoji 裝飾的欄位名稱
   - 實際資料庫是簡潔的英文名稱
   - 導致寫入失敗

3. **權限問題（404 錯誤）**
   - Integration 無法存取現有資料庫
   - 需要手動在 Notion UI 分享，或重新創建
   - 選擇重新創建（更簡潔）

### 解決方案選擇

**為什麼選擇重新創建而不是修復權限？**

1. **速度更快** — 自動化創建 + 自動連結
2. **結構更乾淨** — 欄位設計對齊程式碼邏輯
3. **避免手動操作** — 不需要在 Notion UI 點來點去
4. **測試資料遷移** — 驗證腳本的可移植性

### 程式碼修改重點

**notion_client.py 關鍵修改：**

```python
# 修改前
{"property": "🏦 銀行", "select": {"equals": bank_name}}
{"property": "💰 應繳金額", "number": {"equals": amount}}

# 修改後
{"property": "銀行名稱", "select": {"equals": bank_name}}
{"property": "金額", "number": {"equals": amount}}
```

```python
# 修改前
new_page_props = {
    "帳單": {"title": [{"text": {"content": title_text}}]},
    "🏦 銀行": {"select": {"name": bank_name}},
    "💰 應繳金額": {"number": amount}
}

# 修改後
new_page_props = {
    "Name": {"title": [{"text": {"content": title_text}}]},
    "銀行名稱": {"select": {"name": bank_name}},
    "金額": {"number": amount}
```
